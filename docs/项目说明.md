# 项目说明文档

## 一、项目背景

本项目是《Web 应用开发》课程的期末综合训练作业，由 6 人小组共同完成。

在当今移动互联网时代，休闲小游戏因其简单易上手、随时可以畅玩的特点受到广大用户喜爱。我们小组以"森林"为主题，结合 Web 前后端技术，开发了一款浏览器可直接运行的 2D 跑酷游戏——**森林酷跑**。

---

## 二、项目目标

| 目标 | 说明 |
|------|------|
| 技术学习 | 综合运用 Flask、MySQL、HTML5 Canvas 等课堂所学技术 |
| 功能完整 | 实现用户系统、游戏系统、成绩系统三大模块的完整闭环 |
| 协作开发 | 通过 Git 多人协同开发，模拟真实工程团队协作流程 |

## 三、功能模块介绍

### 3.1 用户系统
- **注册**：填写用户名、昵称、密码创建账号；密码使用 `werkzeug` 库进行 PBKDF2 哈希加密存储
- **登录**：通过用户名和密码验证登录，登录状态通过 Flask Session 维持
- **个人中心**：查看个人最高分、总金币数、游戏次数、历史成绩列表

### 3.2 游戏系统
- **Canvas 渲染**：游戏画面全部通过 HTML5 Canvas API 绘制，实现 60fps 流畅动画
- **跑酷核心**：玩家角色自动向右跑动，通过空格键/↑键跳跃（支持二段跳）躲避障碍物
- **碰撞检测**：使用 AABB（轴对齐包围盒）算法检测玩家与障碍物、金币的碰撞
- **分数系统**：跑动距离自动加分，收集金币额外加分，难度随时间提升

### 3.3 成绩系统
- **自动保存**：每局游戏结束后，前端通过 AJAX 提交分数、金币、距离到后端
- **排行榜**：查询各用户最高分，取全服 TOP 10 展示，前三名有领奖台特效
- **历史记录**：个人中心可查看所有历史成绩

### 3.4 管理后台
- **用户管理**：查看所有注册用户列表
- **公告管理**：发布和删除游戏公告
- **数据统计**：显示总用户数、今日游戏次数、历史最高分等概览

## 四、技术实现思路

### 后端架构
采用 Flask 蓝图（Blueprint）将项目分为 4 个模块：
- `auth`：认证模块（登录/注册/登出）
- `game`：游戏模块（页面渲染）
- `score`：成绩模块（提交/排行榜）
- `admin`：管理模块

### 数据库设计
- `users` 表：存储用户基础信息
- `scores` 表：存储每次游戏成绩，通过 `user_id` 外键关联用户
- `notices` 表：系统公告
- `skins` 表：角色皮肤（扩展）

### 游戏实现
使用浏览器原生 `requestAnimationFrame` 驱动游戏主循环，每帧执行：
1. 更新玩家物理（重力/速度/位置）
2. 生成/移动障碍物和金币
3. 碰撞检测
4. 重绘 Canvas

## 五、项目亮点

1. **纯原生 Canvas 实现**，不依赖任何游戏框架，技术透明度高，方便学习
2. **蓝图模块化架构**，每位成员负责独立模块，代码解耦，合并冲突少
3. **二段跳机制**，增加游戏操作深度
4. **速度动态提升**，游戏难度随时间自动加大，体验更有挑战性
5. **Forest 主题视觉设计**，绿色调、卡通风格，美观统一
6. **全代码中文注释**，适合学习和演讲展示
