# 功能模块图说明

本文档说明功能模块图应包含的模块和层次结构，供组长绘制系统功能模块图时参考。

---

## 一、功能模块总览

```
森林酷跑游戏系统
│
├── 1. 用户管理模块
├── 2. 游戏管理模块
├── 3. 成绩管理模块
├── 4. 趣味扩展模块（可选）
└── 5. 系统管理模块
```

---

## 二、用户管理模块

**入口页面**：登录页、注册页、个人中心

| 功能 | 说明 | 负责成员 |
|------|------|---------|
| 用户注册 | 输入用户名、昵称、密码；前端校验+后端入库 | 成员三 |
| 用户登录 | 用户名+密码登录，生成 Session | 成员三 |
| 用户登出 | 清除 Session，跳转登录页 | 成员三 |
| 个人信息展示 | 显示昵称、头像、注册时间 | 成员三/成员四 |
| 修改昵称 | （扩展）允许用户修改显示昵称 | 成员三 |

---

## 三、游戏管理模块

**入口页面**：游戏页（game.html）

| 功能 | 说明 | 负责成员 |
|------|------|---------|
| 游戏初始化 | Canvas 绘制、变量重置、遮罩显示 | 成员一 |
| 玩家跳跃 | 空格/↑键触发，支持二段跳 | 成员一 |
| 玩家下滑 | ↓键触发，躲避飞行障碍（待开发）| 成员一 |
| 障碍物生成与移动 | 随机生成，从右向左滚动，速度随时间加快 | 成员一 |
| 金币生成与收集 | 随机生成，碰到即收集加分 | 成员一 |
| 碰撞检测 | 玩家与障碍物碰撞触发游戏结束 | 成员一 |
| 分数实时计算 | 每帧加基础分，收金币加额外分 | 成员一 |
| 游戏暂停/继续 | P 键切换暂停状态 | 成员一 |
| 游戏结算显示 | 游戏结束弹出结算面板展示本局数据 | 成员一/成员二 |
| 图片素材替换 | 将矩形角色替换为真实游戏图片 | 成员二 |
| 音效添加 | 跳跃音、金币音、游戏结束音 | 成员二 |
| 粒子特效 | 收集金币时的粒子爆炸效果 | 成员二 |

---

## 四、成绩管理模块

**入口页面**：游戏结束后自动提交、排行榜页、个人中心页

| 功能 | 说明 | 负责成员 |
|------|------|---------|
| 成绩自动提交 | 游戏结束后 AJAX 提交分数到后端 | 成员一/成员三 |
| 成绩入库 | Flask 接口接收，写入 scores 表 | 成员三/成员四 |
| 全服排行榜 | 查询各用户最高分 TOP 10 | 成员四 |
| 个人历史成绩 | 查询当前用户所有成绩记录 | 成员四 |
| 个人统计 | 计算总金币、最高分、游戏次数等 | 成员四 |
| 分页显示 | 排行榜支持分页（扩展） | 成员四 |

---

## 五、趣味扩展模块（可选，时间充裕时实现）

| 功能 | 说明 | 负责成员 |
|------|------|---------|
| 皮肤系统 | 根据最高分解锁不同角色皮肤 | 成员二/成员四 |
| 道具系统 | 无敌星、加速靴等游戏道具 | 成员一/成员二 |
| 关卡系统 | 不同难度关卡，障碍物类型多样 | 成员一 |
| 好友排名 | 查看好友之间的排名（扩展）| 成员三/成员四 |
| 分享功能 | 分享截图或成绩到社交平台 | 成员二/成员五 |

## 六、系统管理模块

**入口页面**：后台管理页（admin.html）

| 功能 | 说明 | 负责成员 |
|------|------|---------|
| 用户列表查看 | 展示所有注册用户，支持搜索 | 成员三 |
| 用户封禁/解封 | 管理违规用户（扩展）| 成员三 |
| 公告发布 | 发布游戏公告，首页展示 | 成员三 |
| 公告删除 | 删除过期公告 | 成员三 |
| 数据统计面板 | 总用户数、今日游戏次数、历史最高分 | 成员三/成员四 |
| 成绩管理 | 查看/删除异常成绩记录（扩展）| 成员四 |

## 七、绘制功能模块图建议

组长在绘制功能模块图时，推荐使用以下工具：

- **ProcessOn**（在线）：https://www.processon.com
- **draw.io**（在线/本地）：https://app.diagrams.net
- **Visio**（本地）：适合有 Office 的同学

**图形规范**：
- 顶层：系统名称（矩形）
- 第二层：5 大模块（矩形，使用不同颜色区分）
- 第三层：具体功能点（矩形）
- 连接线：父子关系用实线，扩展功能用虚线
