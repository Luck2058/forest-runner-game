# 森林酷跑项目 — 任务分工与协作指南

**项目名称**：《森林酷跑》——基于 Flask \+ MySQL 的 Web 休闲跑酷小游戏系统

**项目定位**：类似地铁跑酷 / 熊出没快跑的 2D Web 跑酷小游戏。玩家登录后进入游戏，控制角色跳跃、下滑或左右移动，躲避障碍物，收集金币和道具，游戏结束后系统记录分数、距离、金币数量，并展示排行榜和个人历史成绩。

**技术栈**：Flask 3 \+ MySQL \+ HTML5 Canvas \+ JavaScript
*开发阶段可使用 SQLite 快速测试，最终展示和文档以 MySQL 为准。*

**仓库地址**：[https://github\.com/Luck2058/forest\-runner\-game](https://github.com/Luck2058/forest-runner-game)

**文档更新**：2026\-06\-03

---

**📌 选题依据**：课程要求休闲小游戏具备登录功能、记录玩家成绩，并体现趣味性和可玩性；成果展示需要提交 Word 文档，内容包括功能模块图和运行截图。

# 十二、冲突处理规则

- 如果需要修改别人负责的文件，先在群里说明，征得同意后再改。

- 如果 Git 出现冲突（CONFLICT），不要乱点确认，先截图发群里，由组长或对应模块负责人一起处理。

- 涉及公共文件如 base\.html、style\.css、config\.py、README\.md，由组长统一确认后再改。

- 合并冲突时，保留双方有价值的改动，删除冲突标记（\<\<\<\<\< / ====== / \>\>\>\>\>）后再提交。
